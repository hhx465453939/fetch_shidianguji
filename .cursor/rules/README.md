# Cursor Rules 文档

本项目包含了一套完整的 Cursor Rules 配置，用于指导 AI 助手更好地理解和处理识典古籍获取工具项目。

## 规则文件概览

### 1. 项目结构规则 (`project-structure.mdc`)
- **应用范围**: 始终应用 (`alwaysApply: true`)
- **内容**: 项目核心文件说明、技术栈、目录结构
- **作用**: 帮助 AI 快速理解项目整体架构和文件关系

### 2. Python 编码规范 (`python-coding-standards.mdc`)
- **应用范围**: 所有 Python 文件 (`globs: *.py`)
- **内容**: 编码规范、命名约定、文档字符串、错误处理
- **作用**: 确保代码质量和一致性

### 3. 网页抓取模式 (`web-scraping-patterns.mdc`)
- **应用范围**: 所有 Python 文件 (`globs: *.py`)
- **内容**: 识典古籍网站结构、内容选择器、请求处理模式
- **作用**: 指导网页抓取和内容解析的最佳实践

### 4. 错误处理规范 (`error-handling.mdc`)
- **应用范围**: 所有 Python 文件 (`globs: *.py`)
- **内容**: 异常处理原则、网络请求错误处理、数据验证
- **作用**: 确保代码的健壮性和可靠性

### 5. 配置管理 (`configuration-management.mdc`)
- **应用范围**: Python 文件、环境文件、文档 (`globs: *.py,*.env*,*.md`)
- **内容**: 环境变量配置、配置验证、动态配置
- **作用**: 统一配置管理策略

### 6. 测试指南 (`testing-guidelines.mdc`)
- **应用范围**: Python 文件和测试文件 (`globs: *.py,test_*.py,*_test.py`)
- **内容**: 单元测试、集成测试、测试数据管理、性能测试
- **作用**: 确保代码质量和测试覆盖率

### 7. 古籍文本处理 (`ancient-text-processing.mdc`)
- **应用范围**: Python 文件和 Markdown 文件 (`globs: *.py,*.md`)
- **内容**: 古籍文本清理、格式化、内容验证、元数据管理
- **作用**: 专门针对古籍内容处理的特殊需求

### 8. 开发工作流 (`development-workflow.mdc`)
- **应用范围**: 始终应用 (`alwaysApply: true`)
- **内容**: 开发流程、代码质量保证、项目维护、部署发布
- **作用**: 指导整个开发生命周期

## 使用说明

### 自动应用规则
- `alwaysApply: true` 的规则会在每次 AI 交互时自动加载
- 这些规则提供项目的基础上下文和通用指导原则

### 文件特定规则
- `globs` 规则只在处理匹配的文件类型时应用
- 例如，Python 编码规范只在编辑 `.py` 文件时生效

### 手动应用规则
- 没有 `alwaysApply` 或 `globs` 的规则需要手动引用
- 可以通过描述 (`description`) 字段来触发

## 规则维护

### 更新规则
1. 直接编辑对应的 `.mdc` 文件
2. 确保 frontmatter 格式正确
3. 测试规则是否按预期工作

### 添加新规则
1. 创建新的 `.mdc` 文件
2. 设置适当的 frontmatter 配置
3. 更新此 README 文档

### 规则优先级
1. 文件特定规则 (`globs`) 优先级最高
2. 始终应用规则 (`alwaysApply`) 次之
3. 手动应用规则优先级最低
